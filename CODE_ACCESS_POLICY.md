# Code Access Policy

## Purpose

This repository is published for portfolio, educational, and technical-review purposes.

## What is open for review

- Application architecture and implementation patterns
- API design, fallback strategy, and deployment structure
- Frontend integration and product workflow code
- ML pipeline orchestration code (training/inference logic)

## Restricted / sensitive assets

The following are intentionally excluded and must not be expected in this public repo:

- Private credentials, API keys, and access tokens
- Proprietary or licensed datasets
- Confidential model artifacts/checkpoints not cleared for sharing
- User PII, customer/internal logs, and operational secrets

## Usage expectations

- You may read and learn from the code.
- You may adapt patterns for personal learning or non-commercial experimentation.
- Do not claim this work as your own.
- Do not use branding, project identity, or screenshots to imply endorsement.

## Commercial / production reuse

For commercial or production reuse, contact the repository owner for explicit permission and review requirements (security, compliance, attribution, and licensing scope).

## Security and disclosure

If you discover a security issue, please disclose privately to the repository owner before publishing details.
