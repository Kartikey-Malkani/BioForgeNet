# Contributing

Thanks for your interest in BioForgeNet.

## Scope

This repository is primarily maintained as a project showcase and technical portfolio artifact.

## Ways to contribute

- Open issues for bugs, documentation clarity, and reproducibility gaps.
- Propose improvements through pull requests with clear rationale and test notes.
- Suggest product/UI improvements with screenshots or concrete acceptance criteria.

## Pull request checklist

- Keep changes focused and minimal.
- Preserve existing architecture and naming conventions unless change is intentional.
- Include a short "What changed" + "Why" summary in the PR description.
- Validate modified paths locally before submitting.

## Branch + commit guidance

- Branch naming: `feature/<topic>`, `fix/<topic>`, `docs/<topic>`
- Commit style: concise, action-oriented
  - Example: `Fix booking fallback response handling`
  - Example: `Add model-status startup diagnostics`

## Security and sensitive data

Do not commit:

- API keys, tokens, credentials, or secrets
- private datasets and raw sensitive artifacts
- internal customer data or logs

If you discover a security issue, disclose privately to the owner first.
